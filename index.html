Process

HOW I WORK

I am currently enrolled in SEG3125 — Analysis and Design of User Interfaces at the University of Ottawa, actively learning design methodology and user-centered principles. Key references I'm using: Nielsen Norman Group (NN/g) and the Interaction Design Foundation. Each project this semester taught me a different stage of the design process — here's what I took away from each one.

01

Structure Around the User

Service Website — Fernwood Hair Co.

Designing a salon site taught me information architecture: organizing everything into Home, Services, Booking and Contact around the one thing a visitor actually comes to do — book an appointment. I learned to build clear visual hierarchy, consistent navigation, and a responsive layout with React and Bootstrap so the experience holds up on any screen.

02

Storyboard Before Coding

Memory Game — Pairadise

I storyboarded two complete design directions — start, play and win screens for each — before writing a single line of code, then kept the strongest ideas from both. Building the game itself was a lesson in feedback: every flip, match and win needs an instant visual and sound response. Nielsen's "visibility of system status," in practice.

03

Sweat the Words and the Flow

E-Commerce Site — Le Levain

An artisanal bakery-supply shop taught me that microcopy is part of the interface. I wrote every string in one consistent writer-to-reader voice, designed a checkout that keeps friction low, and closed the loop with a built-in user survey — because a design isn't finished until real users tell you how it went.

04

Make Data Speak Clearly

Analytics Dashboard — Grocery Prices

Visualizing two years of grocery prices with Recharts taught me to match the chart to the question: line charts for trends over time, bar charts for comparing items in a single month. I also shipped it fully bilingual (EN/FR) with locale-aware currency and date formatting — designing for both of Canada's official languages from the start.
